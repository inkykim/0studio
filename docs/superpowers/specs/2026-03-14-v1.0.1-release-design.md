# v1.0.1 Release Design

**Date:** 2026-03-14
**Goal:** Ship a Homebrew-distributable macOS build with cloud sync for single users, hiding team/collaboration and payment features behind compile-time feature flags.

## 1. Feature Flag System

### Implementation

New file `src/lib/features.ts`:

```ts
export const features = {
  team: import.meta.env.VITE_FEATURES_TEAM === 'true',
  payments: import.meta.env.VITE_FEATURES_PAYMENTS === 'true',
} as const;
```

### Environment Configuration

**`.env.development`** (internal testing — all features enabled):
```
VITE_FEATURES_TEAM=true
VITE_FEATURES_PAYMENTS=true
```

**`.env.production`** (shipped build — team/payments hidden):
```
VITE_FEATURES_TEAM=false
VITE_FEATURES_PAYMENTS=false
```

Vite statically replaces `import.meta.env.*` at build time, so hidden features are tree-shaken from the production bundle.

## 2. UI Changes Per Feature Flag

### When `features.team === false`

| Component | Change |
|-----------|--------|
| `WelcomePanel.tsx` | Hide "Shared with you" section. Gate the `useEffect` that fetches shared projects + shows toast notifications so it only runs when `features.team` is true. Keep "Your cloud projects." |
| `VersionControl.tsx` | Hide `<TeamPresencePanel />` and `<CommitPresenceAvatars />` |
| `Settings.tsx` (Project tab) | Hide Team Members section (member list, invite form, permissions legend). Keep Project Info. Reword "Enable collaboration" to "Enable cloud sync" — update all related strings: button text, description text, success toast, section header, and icon. |
| `CloudSyncContext.tsx` | Wrap presence `joinProject`/`leaveProject` calls in `features.team` check |
| `VersionControlContext.tsx` | Gate all `updatePresenceCommit()` calls (9 call sites) — wrap each in `if (features.team)` |
| `App.tsx` | Conditionally render `PresenceProvider` — when `features.team === false`, render a no-op passthrough that provides the context with empty defaults so `usePresence()` never throws |

### When `features.payments === false`

| Component | Change |
|-----------|--------|
| `Auth.tsx` (UserMenu) | Hide "Plans & Billing" dropdown item |
| `Settings.tsx` (Account tab) | Hide "Plan & Billing" section |
| `App.tsx` | Remove `/dashboard` and `/checkout` routes (redirect to `/`) |
| `AuthContext.tsx` | Skip `loadPaymentPlan` API call entirely — set `paymentPlan` to `null` and `paymentPlanLoaded` to `true` immediately |

### Unchanged (always active)

- Auth (sign in/up/out, Google OAuth)
- Cloud sync (push/pull to S3)
- Project registration (reworded for solo use)
- Local version control (commits, branches, tree)
- Gallery mode (compare up to 4 versions)
- Export model
- File watching, drag & drop

## 3. Backend Deployment

**Platform:** Railway (auto-deploys from GitHub, manages env vars, free/cheap tier)

### Steps

1. Create Railway project linked to the repo. **Set root directory to `backend/`** in Railway project settings so it detects the correct `package.json` and runs `npm start`.
2. Configure env vars: `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`, `AWS_REGION`, `S3_BUCKET_NAME`, `SUPABASE_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `STRIPE_SECRET_KEY`, `STRIPE_WEBHOOK_SECRET`, `FRONTEND_URL`. Railway automatically sets `PORT`.
3. Railway runs `npm start` (production entry point).
4. Get production URL (e.g., `https://0studio-backend.up.railway.app`).
5. **Update `VITE_BACKEND_URL` in `.env.production`** to the new Railway URL. Important: only update this AFTER verifying the Railway deployment is working.
6. Update Stripe webhook URL in the Stripe dashboard to point to the new Railway URL (`https://<railway-url>/api/stripe/webhook`).

### CORS Configuration

Electron's `file://` protocol sends `Origin: null` in HTTP requests. Update `backend/server.js` CORS config to use a function-based origin check:

```js
app.use(cors({
  origin: (origin, callback) => {
    // Allow requests with no origin (Electron file://, curl, server-to-server)
    // and the configured frontend URL
    const allowed = [process.env.FRONTEND_URL || 'http://localhost:5173'];
    if (!origin || origin === 'null' || allowed.includes(origin)) {
      callback(null, true);
    } else {
      callback(new Error('Not allowed by CORS'));
    }
  },
  credentials: true,
}));
```

## 4. Packaging & Distribution

**Version:** `1.0.1` (bump in `package.json`)

**Build targets:** macOS DMG + ZIP for x64 and arm64 (existing electron-builder config)

**Build command:** `npm run electron:dist`

**Distribution flow:**
1. Build production artifacts locally
2. Create GitHub Release `v1.0.1`
3. Upload DMG/ZIP artifacts to the release
4. Update Homebrew formula in `inkykim/homebrew-0studio` tap with new URLs and SHA256 hashes

**Homebrew formula:** Cask-style formula that downloads the ZIP from GitHub Releases. Install command: `brew install --cask inkykim/0studio/0studio`. Includes `caveats` block for Gatekeeper bypass instructions:

```
To open 0studio for the first time (unsigned app):
  Right-click the app → Open, or run:
  xattr -cr /Applications/0studio.app
```

**App metadata (unchanged):**
- App ID: `com.rhinostudio.app`
- Product name: `0studio`
- Category: `public.app-category.developer-tools`
